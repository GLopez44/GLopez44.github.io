Memoria RAM para Servidores

Tarjetas RAM de Servidor

La memoria RAM (Random Access Memory) es un componente esencial en cualquier sistema informático, y en el caso de los servidores, su importancia es aún mayor. A diferencia de la memoria RAM utilizada en computadoras personales, la RAM de servidor está diseñada para ofrecer mayor estabilidad, fiabilidad y capacidad de corrección de errores, características que se cumplen a través de las tenologías ECC con su corrección de errores y las técnicas de interleaving y dual/quad channel.

Tipos de Memoria RAM de Servidor

Existen distintos tipos de memoria RAM utilizadas en servidores, entre las cuales destacan RDIMM (Registered DIMM) y LRDIMM (Load-Reduced DIMM).

RDIMM (Registered DIMM)

La memoria R-DIMM o RDIMM (Registered Dual In-Line Memory Module) es un tipo de módulo de memoria que cuenta con un registro o buffer que actúa como un intermediario entre los chips de memoria y el controlador de memoria. Esto ayuda a estabilizar y mejorar la señal antes de enviarla al controlador, lo que la hace ideal para servidores debido a su gran fiabilidad.

LRDIMM (Load-Reduced DIMM)

La memoria LRDIMM reduce la carga del bus de memoria mediante su función de carga-descarga con CCL. Se emplea en servidores de alta capacidad para mejorar la estabilidad y rendimiento en sistemas con múltiples módulos de memoria.

Tecnología ECC

La memoria ECC (Error-Correcting Code) detecta y corrige errores de memoria, lo que la hace ideal para entornos como servidores y centros de datos. Este tipo de memoria ayuda a prevenir la corrupción de datos, aumentando la fiabilidad del sistema y reduciendo las posibilidades de fallos críticos. Su funcionamiento se basa en la detección de bits erróneos y su corrección antes de que lleguen al procesador.

La siguiente imagen muestra un diagrama de fallo de la RAM ECC, donde se puede apreciar como tras haber un periodo de tiempo sin fallo (MUT/MTTF) ocurre un fallo (empiza el MDT) la propia RAM se encarga de detectarlo y solventar el problema (MTTR) y vuelve a su funcionamiento habitual.

Métrica Descripción
MTTF (Mean Time To Failure) Tiempo medio hasta el primer fallo del sistema. En sistemas reparables, es igual al MUT. En sistemas no reparables, es igual al MTBF.
MTBF (Mean Time Between Failures) Tiempo medio entre fallos, mide la fiabilidad del sistema. Se calcula como MTBF = MTTF + MDT.
MDT (Mean Down Time) Tiempo que el sistema está fuera de servicio tras un fallo. Incluye el tiempo de detección y reparación.
MTTR (Mean Time To Repair) Tiempo medio necesario para reparar un sistema tras un fallo. Debe ser lo más corto posible para reducir el impacto.

Técnicas de Interleaving y Dual/Quad Channel

El Interleaving es una técnica utilizada en la memoria RAM para mejorar el acceso a los datos, reduciendo la latencia y aumentando el rendimiento. En lugar de leer o escribir toda la información desde un solo módulo, los datos se distribuyen en múltiples módulos o bancos de memoria, permitiendo que el procesador acceda a ellos en paralelo.

  • Interleaving de banco: Se usa dentro de un solo módulo de RAM, alternando datos entre diferentes bancos internos para acelerar las operaciones.
  • Interleaving de canal (Dual/Quad Channel): Distribuye la carga de trabajo entre múltiples módulos físicos de RAM, permitiendo que el procesador acceda a más datos simultáneamente.
    • Dual Channel: El Dual Channel es una tecnología que permite que dos módulos de RAM trabajen juntos para duplicar el ancho de banda de la memoria.
    • Quad Channel: El Quad Channel es una tecnología que permite que cuatro módulos de RAM trabajen simultáneamente, cuadruplicando el ancho de banda de la memoria.
